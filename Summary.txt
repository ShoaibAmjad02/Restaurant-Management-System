# Restaurant Management System — Project Summary

## 1. Project Overview

A full-stack restaurant management and QR-based food ordering system built with Django, MySQL, and Bootstrap. The platform supports online ordering, QR-based table ordering, kitchen display management, invoice generation, loyalty program, time-based offers/discounts, and an administrative dashboard — all unified in a single system.

Key capabilities:
- Online customers: register, log in, browse menu, place orders, earn loyalty points, track orders in real time
- Dine-in customers: scan table QR code, access digital menu, order without waiter assistance
- Kitchen staff: receive orders instantly via kanban dashboard, update statuses, sound notifications
- Administrators: manage products, tables, QR codes, kitchen users, offers, deals, loyalty cards, revenue analytics, database backups

### 4.5 Authentication

- **Custom User Model**: `users.User` — email-based (no username), `USERNAME_FIELD = "email"`
- **Backends**: `ModelBackend` + `allauth.account.auth_backends.AuthenticationBackend`
- **Login URL**: `food-delivery:food_delivery_login` (`/food-delivery/login/`)
- **Login Flow**: POST email + password to `users:login`; auto-creates LoyaltyCard for customers; handles `?next=` parameter; redirects staff→admin, kitchen→kitchen, customers→home (or loyalty welcome)
- **Registration**: POST name + email + password to `users:register`; redirects to login
- **Logout**: `/users/logout/`
- **Password Policy**: Argon2 hashing; minimum length, common password, numeric, and similarity validators
- **allauth**: Email-only, mandatory verification (`ACCOUNT_EMAIL_VERIFICATION = "mandatory"`), custom signup form

1. User adds items to cart (localStorage)
2. Clicks "Checkout" in minicart
3. Payment modal opens with order summary

## 11. Authentication Flow

### 11.1 Login Redirects
- `LOGIN_URL` configured as `"food-delivery:food_delivery_login"` (`/food-delivery/login/`)
- All `@login_required` decorators and `LoginRequiredMixin` use this custom login page
- Django's default `/accounts/login/` is never used

### 11.2 Protected Routes
- `/users/restaurant-detail/` — `@login_required(login_url='food-delivery:food_delivery_login')`
- `/users/checkout/` — `@login_required` + `@require_POST`
- `/users/loyalty-card/` — manual auth check with message + redirect
- `/users/kitchen/dashboard/` — `@login_required` + `is_kitchen` check
- `/users/dashboard/` — `@staff_member_required`
- `/users/products/` — `@staff_member_required`
- `/users/tables/` — `@staff_member_required`
- All admin pages — `@staff_member_required`
- `UserDetailView`, `UserUpdateView`, `UserRedirectView` — `LoginRequiredMixin`

### 11.3 `?next=` Parameter Handling
- Login view reads `request.GET.get("next")` and `request.POST.get("next")`
- Login template renders hidden `<input name="next">` when `request.GET.next` is present
- After successful login, redirects to `next_url` if provided
- Used by: loyalty card redirect, homepage Add to Cart redirect, deal checkout redirect

Restaurant Management and QR-Based Food Ordering System is a full-stack web application developed using Django, MySQL, HTML, CSS, Bootstrap, JavaScript, and AJAX. The system integrates online food ordering, QR-based table ordering, kitchen management, invoice generation, and administrative control into a single unified platform.

The system supports both online and offline restaurant customers. Online customers can register, log in, browse menu items, place orders, generate invoices, and track order status in real time. Offline customers can scan a unique QR code placed on each restaurant table to access the digital menu, place orders directly from their table, and monitor order progress without requiring waiter assistance.

Each restaurant table is assigned a unique QR code containing a secure table identifier. When scanned, customers are redirected to the restaurant menu where they can add items to their cart, complete checkout, and automatically create an order linked to their table number. A real-time order tracking system allows customers to monitor order statuses such as Pending, Preparing, Ready, and Delivered. Once an order is delivered, the system automatically generates and opens a thermal-style PDF invoice containing a traceable QR code for invoice verification.

A dedicated kitchen dashboard enables kitchen staff to receive new orders instantly, monitor incoming requests, update order statuses, search orders by table number, invoice number, or order number, and manage the complete food preparation workflow. Sound notifications alert kitchen staff whenever a new order is received.

The administrative dashboard provides centralized control over menu management, category management, table and QR code management, kitchen user management, invoice searching, revenue monitoring, order tracking, customer management, and database backup functionality. Administrators can generate QR codes for restaurant tables, manage kitchen staff accounts, filter revenue reports using date ranges, and monitor restaurant performance through an intuitive and professional interface.

The system improves operational efficiency by automating restaurant workflows, eliminating manual order handling, reducing service delays, enhancing customer experience, enabling contactless ordering, and providing complete real-time monitoring and management capabilities through a secure and user-friendly platform.
